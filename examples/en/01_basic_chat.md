# 01_basic_chat — Single-turn / Multi-turn Chat

This example demonstrates how to call a locally deployed Tencent Hunyuan Hy3 model via the OpenAI-compatible API to perform single-turn and multi-turn chat. It highlights:

- Initializing the client via environment variables;
- Single-turn chat: send one `user` message and print the reply;
- Multi-turn chat: pass a `system` / `user` / `assistant` / `user` history in `messages` and call again, illustrating context continuity.

## Introduction

Hy3 provides an OpenAI-compatible `/v1/chat/completions` endpoint that can be called directly with the `openai` SDK. This example uses the recommended params `temperature=0.9` and `top_p=1.0`, and disables the reasoning process via `extra_body={"chat_template_kwargs": {"reasoning_effort": "no_think"}}` to get a direct answer.

## Complete Request Code

The following code is identical to `01_basic_chat.py`:

```python
"""Hy3 Example 01: Single-turn / Multi-turn Chat.

Demonstrates:
1. Single-turn chat: send one user message and print the reply.
2. Multi-turn chat: pass a system / user / assistant / user history and call again.

Works with local vLLM/SGLang or cloud TokenHub (set HY3_BASE_URL / HY3_API_KEY).
"""

from __future__ import annotations

import os
import sys

# Allow `python examples/en/01_basic_chat.py` from repo root
sys.path.insert(0, os.path.abspath(os.path.join(os.path.dirname(__file__), "..")))

from common import chat_completion, extract_reasoning_and_content, get_config, make_client  # noqa: E402


def single_turn(client):
    print("=" * 60)
    print("[Single-turn Chat]")
    print("=" * 60)

    messages = [
        {"role": "user", "content": "用一句话介绍腾讯混元 Hy3 模型。"},
    ]
    response = chat_completion(client, messages, reasoning="no_think")
    _, content = extract_reasoning_and_content(response.choices[0].message)

    print(f"User: {messages[0]['content']}")
    print(f"Assistant: {content}")
    if response.usage:
        print(
            f"Usage: prompt={response.usage.prompt_tokens} "
            f"completion={response.usage.completion_tokens} "
            f"total={response.usage.total_tokens}"
        )
    print()


def multi_turn(client):
    print("=" * 60)
    print("[Multi-turn Chat]")
    print("=" * 60)

    messages = [
        {"role": "system", "content": "你是一个简洁友好的中文助手。"},
        {"role": "user", "content": "Hy3 的上下文长度是多少？"},
        {"role": "assistant", "content": "Hy3 的上下文长度为 256K tokens。"},
        {"role": "user", "content": "那它的激活参数量是多少？"},
    ]
    response = chat_completion(client, messages, reasoning="no_think")
    _, content = extract_reasoning_and_content(response.choices[0].message)

    for msg in messages:
        print(f"{msg['role']}: {msg['content']}")
    print(f"assistant: {content}")
    print()


def main():
    cfg = get_config()
    print(f"Connecting to {cfg['base_url']}  model={cfg['model']}")
    client = make_client()
    single_turn(client)
    multi_turn(client)


if __name__ == "__main__":
    main()
```

## Complete Response Parsing

`client.chat.completions.create(...)` returns a `ChatCompletion` object with the following key fields:

### 1. `response.id`

The unique identifier (string) of this completion request, e.g. `chatcmpl-xxxx`. Useful for log tracing and troubleshooting.

```python
print(response.id)
# Example: chatcmpl-9f2c1a8b7e6d4...
```

### 2. `response.choices[0].message.role`

The role of the model's reply for this turn, always `"assistant"`.

```python
print(response.choices[0].message.role)
# assistant
```

### 3. `response.choices[0].message.content`

The text content generated by the model — the field most commonly accessed in everyday use.

```python
print(response.choices[0].message.content)
# Example: Hy3 is a 295B-parameter MoE large model released by Tencent's Hunyuan team, with about 21B activated parameters.
```

> `choices` is a list (usually of length 1, corresponding to `n=1`); use `response.choices[0]` to get the first candidate, then access its `.message`.

### 4. `response.usage`

The token usage statistics for this call, with three integer fields:

| Field | Meaning |
| --- | --- |
| `prompt_tokens` | Number of input tokens (system + history + current user turn) |
| `completion_tokens` | Number of tokens generated by the model |
| `total_tokens` | Sum of the two above |

```python
usage = response.usage
print(usage.prompt_tokens)      # input token count
print(usage.completion_tokens)  # output token count
print(usage.total_tokens)       # total token count
```

### Access Path Quick Reference

```
response
├── id                                # request ID
├── choices[0]
│   ├── message
│   │   ├── role                      # "assistant"
│   │   └── content                   # model reply text
│   └── finish_reason                 # stop reason, e.g. "stop"
└── usage
    ├── prompt_tokens                 # input token count
    ├── completion_tokens             # output token count
    └── total_tokens                  # total token count
```
