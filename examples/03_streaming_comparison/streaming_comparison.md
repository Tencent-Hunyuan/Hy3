# 示例 3：流式 vs 非流式 —— 性能对比

## 概述

对比同一个请求在流式和非流式模式下的性能差异，重点关注：

- **首 token 延迟（TTFT, Time-To-First-Token）**：用户看到第一个字需要多久
- **端到端总耗时**：请求发出到全部内容返回的总时间
- **输出内容一致性**：两种模式生成的内容是否一致

---

## 测试配置

|项目|值|
|:-|:-|
|Prompt|"请用 Python 实现一个 LRU 缓存，包含 get 和 put 方法，并解释时间复杂度。"|
|模型|`hy3`|
|温度|`0.7`|
|推理模式|`no_think`|

---

## 预期对比结果

|指标|非流式 (stream=False)|流式 (stream=True)|
|:-|:-|:-|
|**首 token 延迟**|❌ 无法测量（等待全部完成）|✅ 极低（通常 ~1-2s）|
|**总耗时**|✅ 略低|⚠️ 略高（逐 chunk 传输开销）|
|**用户体验**|等待全部完成才能看到结果|立即开始显示，实时输出|
|**输出内容**|完整字符串|需自行拼接 delta|

---

## 关键指标说明

### 首 token 延迟 (TTFT)

首 token 延迟是衡量模型"响应速度"的核心指标：

- **流式**：TTFT 包括网络传输 + 服务端计算第一个 token 的时间
- **非流式**：不存在"首 token"概念，用户只能等到全部生成结束

在交互式场景（聊天、客服）中，**TTFT 比总耗时更重要**——用户宁可先看到内容逐步出现，也不愿等待几秒后突然看到全部内容。

### 端到端总耗时

- **非流式**：一次 HTTP 往返，返回完整的 JSON
- **流式**：多次 HTTP chunk 传输，每次携带一个或几个 token

对于超长文本生成，流式的总耗时可能比非流式高 10-30%，但用户体验远优于非流式。

## 运行结果示例

```text
============================================================
【流式 vs 非流式 性能对比】
============================================================

Prompt: 请用 Python 实现一个 LRU (Least Recently Used) 缓存...

----------------------------------------
1. 非流式调用 (stream=False)
----------------------------------------
总耗时:    12.34s
prompt tokens:  35
completion tokens: 312
总 tokens: 347
输出长度:  1256 字符

----------------------------------------
2. 流式调用 (stream=True)
----------------------------------------
首 token 耗时 (TTFT):  1.45s
总耗时:               14.23s
输出长度:              1256 字符

============================================================
【对比总结】
============================================================

指标                    非流式           流式
-----------------------------------------------
首 token 延迟 (TTFT)     N/A             1.45s
总响应时间               12.34s          14.23s
输出一致性               —               ✓ 一致
内容长度                 1256 字符       1256 字符

--- 关键结论 ---
1. 流式模式下首 token 仅需 1.45s，用户感知响应极快
2. 流式总耗时 (14.23s) 高于非流式 (12.34s)，逐 chunk 传输有额外开销
3. 流式输出与非流式完全一致
```

## 选择建议

|场景|推荐模式|原因|
|:-|:-|:-|
|聊天机器人|✅ 流式|用户期望即时反馈|
|批量文本生成|✅ 非流式|总耗时更短，编程更简单|
|流式渲染 UI|✅ 流式|类似打字机效果，体验佳|
|后端 API 调用|✅ 非流式|编程简单，处理方便|
|实时翻译/语音|✅ 流式|需要低首 token 延迟|
