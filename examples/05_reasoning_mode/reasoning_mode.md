# 示例 5：推理模式对比（快思考 vs 慢思考）

## 概述

Hy3 支持三种推理模式，通过 `reasoning_effort` 参数控制：

| 模式 | 值 | 说明 | 适用场景 |
|------|-----|------|---------|
| **直接回答** | `"no_think"` | 模型直接回答问题，无显式推理过程 | 日常对话、简单问答、翻译 |
| **轻量推理** | `"low"` | 模型进行简短推理，输出 `reasoning_content` | 中等复杂度推理 |
| **深度推理** | `"high"` | 模型进行详细深度推理，输出完整 `reasoning_content` | 数学证明、复杂代码、逻辑题 |

---

## 请求对比

### no_think 模式

```json
{
  "model": "hy3",
  "messages": [{"role": "user", "content": "三个盒子标签都贴错了..."}],
  "temperature": 0.7,
  "top_p": 1.0,
  "reasoning_effort": "no_think"
}
```

### high 模式

```json
{
  "model": "hy3",
  "messages": [{"role": "user", "content": "三个盒子标签都贴错了..."}],
  "temperature": 0.7,
  "top_p": 1.0,
  "reasoning_effort": "high"
}
```

---

## 响应对比

### no_think 响应

```json
{
  "choices": [{
    "message": {
      "role": "assistant",
      "content": "最少只需要拿 1 次。从标有\"混合\"的盒子里拿出一个水果……",
      "reasoning_content": null
    }
  }]
}
```

**无推理过程**，直接返回答案。

### high 响应

```json
{
  "choices": [{
    "message": {
      "role": "assistant",
      "content": "最少只需要拿 1 次……",
      "reasoning_content": "让我们一步步分析：\n1. 三个盒子的标签分别是：苹果、橘子、混合\n2. 所有标签都贴错了……"
    }
  }]
}
```

**包含 `reasoning_content`**，展示完整的推理过程。

---

## Python 代码解析

```python
# 设置推理模式
response = client.chat.completions.create(
    model="hy3",
    messages=[{"role": "user", "content": prompt}],
    extra_body={"reasoning_effort": "high"},  # 可选: no_think, low, high
)

message = response.choices[0].message

# 获取最终回答
print(message.content)

# 获取思考过程（仅 low 和 high 模式有）
reasoning = getattr(message, 'reasoning_content', None)
if reasoning:
    print("思考过程:", reasoning)
```

```text
└──────────────┴──────────────┴──────────────────┴──────────────────────┘
```

---

## 关键要点

1. **`reasoning_effort` 通过 `extra_body` 传入**
2. **`reasoning_content` 字段**仅在 `low` 和 `high` 模式下才有值
3. **性能权衡**：深度推理耗时更长，但复杂任务准确性更高
4. **适合深度推理的场景**：数学证明、逻辑谜题、代码调试、多步规划
5. **适合快思考的场景**：闲聊、翻译、摘要、简单问答
