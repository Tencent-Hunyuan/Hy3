# Using Hy3 with CodeBuddy / WorkBuddy

> 🌐 中文版本： [codebuddy.md](codebuddy.md)

## Introduction

[CodeBuddy](https://www.codebuddy.ai) is an AI-powered full-stack development assistant, and WorkBuddy is its workflow-automation edition. Both support **custom model integration**, so you can configure Hy3 as the backend inference engine for code generation, project scaffolding, and automated workflow orchestration.

## Use Cases

- Full-stack projects generated from scratch (frontend + backend + database)
- AI-driven PR review and code review
- Automated workflows: doc generation, test-case generation, CI/CD scripts

## Requirements

| Item | Requirement |
|:---|:---|
| CodeBuddy version | ≥ 1.0 (latest recommended) |
| Hy3 service | Self-hosted OpenAI-compatible API endpoint |

## Configuration

### CodeBuddy Provider config

Add a custom Provider in CodeBuddy's **Provider settings**:

```yaml
# Or fill it in via the UI config panel
provider:
  name: "Hy3"
  type: "openai-compatible"
  base_url: "https://tokenhub.tencentmaas.com/v1"
  api_key: "your-api-key"
  default_model: "hy3"

  # Optional: default model parameters
  default_params:
    temperature: 0.9
    top_p: 1.0
    max_tokens: 8192
    reasoning_effort: "high"  # Recommended in Agent mode
```

### Recommended parameters per work mode

| Work mode | `reasoning_effort` | `temperature` | `max_tokens` |
|:---|:---|:---|:---|
| Code generation (Chat) | `high` | 0.9 | 4096 |
| Code completion (Complete) | `low` | 0.6 | 1024 |
| PR Review (Agent) | `high` | 0.3 | 8192 |
| Doc generation (Agent) | `low` | 0.9 | 16384 |
| Workflow orchestration (WorkBuddy) | `high` | 0.9 | 8192 |

## End-to-End Demo: Build a REST API Project with CodeBuddy + Hy3

### Core code preview (generated by Hy3)

```python
# main.py
from fastapi import FastAPI
from routes import router

app = FastAPI(
    title="Book Management API",
    description="A book management system built with FastAPI + SQLite",
    version="1.0.0"
)
app.include_router(router, prefix="/api/v1")

# models.py
from pydantic import BaseModel, Field
from typing import Optional
from datetime import datetime

class BookCreate(BaseModel):
    title: str = Field(..., min_length=1, max_length=200)
    author: str = Field(..., min_length=1, max_length=100)
    isbn: str = Field(..., pattern=r"^\d{10}|\d{13}$")
    published_year: Optional[int] = Field(None, ge=1900, le=2026)

class BookResponse(BookCreate):
    id: int
    created_at: datetime
    updated_at: datetime

    class Config:
        from_attributes = True

# routes.py
from fastapi import APIRouter, HTTPException, Query
from models import BookCreate, BookResponse
from database import BookDB

router = APIRouter()
db = BookDB("books.db")

@router.get("/books", response_model=list[BookResponse])
async def list_books(page: int = Query(1, ge=1), size: int = Query(20, le=100)):
    """Get the book list, with pagination support"""
    return await db.list(page, size)

@router.post("/books", response_model=BookResponse, status_code=201)
async def create_book(book: BookCreate):
    """Create a new book"""
    return await db.create(book)

@router.get("/books/{book_id}", response_model=BookResponse)
async def get_book(book_id: int):
    """Get details of a single book"""
    result = await db.get(book_id)
    if not result:
        raise HTTPException(status_code=404, detail="Book not found")
    return result

@router.put("/books/{book_id}", response_model=BookResponse)
async def update_book(book_id: int, book: BookCreate):
    """Update book information"""
    result = await db.update(book_id, book)
    if not result:
        raise HTTPException(status_code=404, detail="Book not found")
    return result

@router.delete("/books/{book_id}", status_code=204)
async def delete_book(book_id: int):
    """Delete a book"""
    deleted = await db.delete(book_id)
    if not deleted:
        raise HTTPException(status_code=404, detail="Book not found")
```

### WorkBuddy automation workflow example

```yaml
# workbuddy-hy3.yaml
name: "PR Review Pipeline"
trigger:
  on: "pull_request"
  branches: ["main", "develop"]

jobs:
  code-review:
    runs-on: "hy3-agent"
    steps:
      - name: "Review code changes"
        provider: "hy3"
        prompt: |
          Review the code changes in this PR:
          1. Check code style and best practices
          2. Identify potential security vulnerabilities
          3. Assess performance impact
          4. Provide improvement suggestions (output in markdown)

      - name: "Generate CHANGELOG"
        provider: "hy3"
        prompt: |
          Based on the changes in this PR, generate a CHANGELOG entry.
```

## Common Notes

| Issue | Cause | Solution |
|:---|:---|:---|
| Provider connection fails | Wrong `base_url` or port | Test service availability with `curl` in the terminal |
| Unstable code quality | `temperature` too high | Lower to `0.3-0.6` in Agent mode |
| WorkBuddy pipeline interrupted | A single step timed out | Increase the timeout or split into smaller steps |
| Incomplete file structure | Not enough tokens in a single generation | Increase `max_tokens` to 8192+ |
| Long-task context lost | An overly long project description exceeds the context | Describe in segments — scaffold first, then refine |
