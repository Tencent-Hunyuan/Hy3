# Hy3 轨迹复盘台手动测试用例

测试地址：<http://127.0.0.1:5173>

测试前提：按 README 启动前端与 API。在线用例还要求后端环境变量已配置，并会消耗托管额度；其余用例可离线完成。

## TC-01：编程循环离线复盘

1. 保持顶部“离线演示”为选中状态。
2. 在“编程循环”卡片点击“开始复盘”。
3. 查看三栏复盘工作区。

预期结果：

- 页面标题为“修复 slugify 的连续分隔符”。
- 首个偏航点为“第 06 步”，编号为 `step-006-repeat-patch`。
- 时间线第 06 步以琥珀色高亮。
- 验收覆盖包含“未满足 / 已通过 / 待确认”三种状态。
- 最小重放计划从 `step-006-repeat-patch` 开始，共三项操作。

## TC-02：证据查看与 JSON 导出

1. 完成 TC-01。
2. 点击“查看关键证据”，或点击任意 `ev-*` 证据标签。
3. 检查证据抽屉的来源、关联步骤、类型和原文。
4. 点击“关闭”，再点击“导出 JSON”。

预期结果：

- 证据抽屉打开后焦点落在“关闭”按钮；按 `Esc` 也能关闭。
- 下载文件名为 `轨迹复盘报告.json`。
- JSON 中 `finding.first_divergence_step_id` 为 `step-006-repeat-patch`。

## TC-03：研究证据漂移与 Markdown 导出

1. 保持“离线演示”。
2. 在“研究证据漂移”卡片点击“开始复盘”。
3. 打开证据 `ev-source-a`，阅读来源摘录后关闭。
4. 点击“导出 Markdown”。

预期结果：

- 首个偏航点为 `step-006-unsupported-causal-leap`。
- 页面指出描述性相关被错误升级为因果结论。
- 下载文件名为 `轨迹复盘报告.md`。
- Markdown 包含“## 最小重放计划”和该偏航步骤编号。

## TC-04：在线 Hy3 分析

1. 点击顶部“在线 Hy3”。
2. 选择“编程循环”并开始复盘。

预期结果：

- 分析期间显示“正在追踪证据边界”，可点击“停止等待”；文案明确说明在线服务端请求可能继续至超时。
- 成功后元数据标记为“在线 Hy3”，并显示耗时与令牌数。
- 面向用户的分析说明、重放操作和验证闸门使用简体中文。
- 如果服务限流，页面显示中文等待秒数和“重试”按钮，不泄露 Key 或上游请求信息。

## TC-05：导入自定义轨迹

1. 点击“导入自定义轨迹”的“选择轨迹文件”。
2. 选择项目中的 `fixtures/coding-loop/input.json`。
3. 出现“已就绪”后点击“分析导入轨迹”。

预期结果：

- 文件通过本地结构校验，文件名显示在卡片中。
- 产品自动使用在线 Hy3 分析。
- 页面不会显示、请求或保存 API Key。

## TC-06：非法导入与移动端

非法导入：

1. 在自定义轨迹卡片选择 `fixtures/manual/invalid-trace.json`。
2. 预期页面显示“文件不符合轨迹复盘任务结构”，且不会开始分析。

移动端：

1. 把浏览器窗口缩到约 390 × 844。
2. 分别打开研究场景和证据抽屉。
3. 预期卡片与报告切换为单栏、页面没有横向滚动，证据抽屉不超出视口。
